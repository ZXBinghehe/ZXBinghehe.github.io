  • instanceof 用于检测是什么类型的对象,返回true或false
    基本语法：
    result=variable instanceof constructor
    person instanceof Object–>变量person是Object吗

    执行环境和作用域

    1 没有块级作用域

2 垃圾收集
javascript具有自动垃圾收集机制，其原理是：找出那些不再继续使用的变量，然后释放其内存，为此，垃圾收集器会按照固定的时间间隔，周期性的执行这一操作。垃圾收集机制可以采取两种方法：

  • 标记清除：
    垃圾收集器在运行时给内存中的所有变量都加上标记，然后，它会去掉环境中的变量和被环境中变量引用的变量的标记，最后清除仍带有标记的变量
  • 引用计数
    跟踪记录每一个值被引用的次数，当这个值的引用次数变成0时，则说明没有办法再访问这个值了，当垃圾收集器下次再运行时，就会释放引用次数为零的值所占用的空间。
    但是这种方式可能会导致循环引用
    3 管理内存
    确保占用最小的内存可以让页面获得更好的性能，而优化内存占用的最佳方式，就是为执行中的代码只保存必要的数据，一旦数据不再有用，就将其值设置为null值来释放内存

javascript操作符

位操作符

后台会发生如下转换过程：64位的数值先被转化为32位，然后再进行位操作，操作完毕后，再将数值转换为64位，这样的过程就可能导致数据缺失，并且在对特殊的NaN和Infinity值进行位操作时，这两个值会被当做0来处理
如果对非数值型数据进行位操作，会先调用Number（）函数进行转换
1.按位非（~)
按位非执行的结果就是返回数值的反码，即负数减1
2.按位与（&）1为真，0为假
3.按位或（|）
4.按位异或（^)两个相同时为假
5.左移<<
左移不会影响操作数的符号位，左移一位相当于乘以2的一次方
6.右移>>右移不会影响操作数的符号位，右移一位相当于除以2的一次方
7.无符号右移>>>负数无符号右移，；有可能会改变符号，并且使右移后的数非常大

布尔操作符

1.逻辑非（！)
2.逻辑与(&&)：如果第一个操作数能够决定结果，则不会对第二个操作数求值
在有一个操作数不是布尔值的情况下，逻辑与操作不一定返回布尔值

  • 如果第一个操作数是对象，则返回第二个操作数
  • 如果第二个操作数是对象，则只有在第一个操作数的求值结果为true时，才会返回该对象
  • 如果两个操作数都是对象，则返回第二个操作数
  • 如果有一个操作数是Null,则返回Null
  • 如果有一个操作数是NaN,则返回NaN
  • 如果有一个操作数是undefined，则返回undefined
    3.逻辑或（||）
    逻辑或也不一定返回布尔值
  • 如果第一个操作数是对象，则返回第一个操作数
  • 如果第一个操作数的求值结果为false，则返回第二个操作数
  • 如果两个操作数都是对象，则返回第一个操作数
  • 如果两个操作数都是Null/undefined/NaN,则返回Null/undefined/NaN
    如果第一个数的操作结果为true，则不会对第二个操作数求值

    乘性操作符

    1.乘法（）如果参与乘性运算的操作数不是数值，会先调用Number（）进行转换 infinity*0=NaN
    2.除法（/)
  • 如果有一个操作数是NaN，则结果是NaN
  • infinity/infinity=NaN
  • 0/0=NaN
    6.求模（%）：取余数

    相等操作符

    1.相等（==）和不相等(!=)-先转换再比较
  • 如果操作数为布尔值，则在比较之前先将其转换为数值，true=1，false=0；
  • 如果有一个操作数位字符串，则会先将字符串转换为数值
  • 如果一个操作数是对象，则调用对象的valueOf（）方法，用到的基本类型值按照前面的规则进行比较
  • Null==undefined—>true;
  • 如果有一个操作数为NaN，则相等操作符返回false，不相等操作符返回true，特别注意NaN==NaN—》false
  • 如果两个操作符都是对象，则比较它们是不是同一个对象
  • 要比较相等性之前，不能将NaN和undefined转换成其他任何值
    2.全等(===)和不全等(!==)-仅比较不转换
  • Null===undefined—>false

    语句

    1.for-in语句

    //对对象 进行遍历

}

  1. with语句：作用是将代码的作用域设置到一个特定的对象中，严格模式下不允许使用with语句

数据结构与算法复习笔记

数据结构的分类

数据结构分为逻辑结构和物理结构

逻辑结构：数据对象中数据元素之间的相互关系。

四大逻辑结构

  • 集合结构：所有数据都在一个集合里，彼此之间无联系
  • 线性结构：所有数据都在串在一条线上，一对一
  • 树形结构：数据结构像一棵树一样，类似于族谱结构
  • 图形结构：数据之间的关系错综复杂，多对多

    物理结构：数据的逻辑结构在计算机中的存储形式。

    物理结构可以分为顺序存储和链式存储
  • 顺序存储：类似于排队，每一个数据有一个固定的位置，某一数据取出后，其位置让出给后面的数据。
  • 链式存储：类似于取号排队，数据按一定顺序排序但可以随意走动，存储位置不固定，但是因为使用链式存储存储数据时，除了存储数据本身外，还要有一定空间存储上一个数据的地址指针，所以链式存储耗费的空间比顺序存储大。

    算法的五个特征

  • 有穷性：算法在执行有限步骤后，自动结束而不会出现无限循环，并且在可接受的时间范围内完成
  • 确定性：算法的每个步骤都具有确定的含义，不会出现二义性
  • 算法在一定条件下，一条执行路径，相同的输入只有有唯一的输出
  • 算法的每个步骤都应精确定义而无歧义
  • 可行性：算法每一步都必须可行，即每一步都能够通过执行有限次数完成

    算法效率（执行时间）的度量方法

  • 事后统计
    缺陷：算法和测试程序时一对一的,费时费力。
  • 事前分析估算方法
    程序运行耗时的影响因素：
    1.算法采用的策略和方案
    2.编译产生的代码质量（编译器）
    3.问题的输入规模
    4.机器执行指令的速度
    抛开计算机的硬件与软件，我们在分析一个算法的运行时间时，重要的是把基本操作的数量和输入模式关联起来。

在比较执行次数时，可以忽略加法常数，与最高次项相乘的常数也不重要
判断一个算法效率时，函数中的常数和其他次要项常常可以忽略，而更应该关注主项（最高次项）的阶数

算法的时间复杂度和空间复杂度（执行次数==时间）

用大写O（）来体现算法时间复杂度的记法，称为大O记法
分析时间复杂度的规律：

  • 用常数1取代运行时间中所有的加法常数
  • 在修改后的运行次数函数中，只保留最高阶项
  • 如果最高项存在且不是1，则去除与这个项相乘的常数
  • 常见的时间复杂度所耗费的时间从小到大依次是：O(1)<O(logn)<(n)<O(nlogn)<O(n^2)<O(n^3)<O(2^n)<O(n!)<O(n^n)

AJAX

什么是AJAX[ˈeɪˌdʒæks]

　　AJAX(Asynchronous Javascript And XML),也就是异步Javascript和XML,用于在不重载整个网页的情况下，通过后台加载数据，实现网页的局部更新。

XMLHttpRequest对象

　　XMLHttpRequest是负责客户端与服务端之间进行通信的对象。AJAX就是将JavaScript和XMLHttpRequset对象放在客户端和服务端之间，当用户在客户端填写数据时，客户端不是等用户填写完所有数据再向服务端发送请求，而是在用户填写完某一项信息时，javascript代码捕获信息并向服务端发送请求，此时，用户并不知道请求的发送，仍然可以进行输入数据，滚动屏幕等操作。XMLHttpRequest对象就是用于向服务端发送请求并处理服务器返回的数据，即XMLHttpRequest对象可以根据需要自行与服务器进行交互。

向服务器发送请求

HTTP

　　HTTP是计算机通过网络进行通信的规则，http是一种无状态协议，无状态是指客户端与服务端之间不建立持久的连接。

客户端向服务端发送请求包含七个步骤：
  1. 建立TCP连接
  2. 客户端发送请求命令
  3. 客户端发送请求头信息
  4. 服务端应答
  5. 服务端发送应答头信息
  6. 服务端向浏览器发送数据
  7. 服务端关闭TCP连接
    GET 和 POST
    　　get:一般用于信息获取；使用URL传递参数，所以不适合向服务端发送密码等私密数据；使用get方法发送信息由数量限制，一般在2000个字符以内。
    　　post：一般用于修改服务器上的资源；使用post方法发送信息无数量限制，因为信息是写在请求体内的。

    向服务器发送请求使用到的方法

    　　向服务器发送请求要用到两个方法：open(method,url,async) 和 send();
    open(method,url,async)
    　　open方法包含三个参数：
  • method：向浏览器请求的方法，post或者get
  • url：请求的数据地址

  • async:是否异步，true代表异步，false代表同步，默认为true

    send(string)

    　　如果使用post方法，像服务端传递的数据作为send方法的参数。

    处理服务端的应答

    服务端的响应

    　　服务端的响应分为三部分：状态码（数字+文字）、响应头和响应体。

    状态码
    • 1xx 信息类
    • 2xx 成功，比如 200 ok
    • 3xx 重定向
    • 4xx 客户端错误，找不到页面，比如404 MOT FOUND
    • 5xx 服务端错误，比如500

      取得响应

      　　request中有各种属性和方法用于获取服务端返回的信息。
      1.readyState属性：
    • 0：open未调用
    • 1：open调用,即以建立与服务器之间的连接
    • 2：请求已被服务端接收
    • 3：请求处理中

    • 4：请求已完成
      2.responseText:服务端返回的数据信息
      3.reponseXML:服务端返回的XML形式的数据
      4.status:状态码(数字)
      5.statusText:状态码（字符串）
      6.getAllResponseHeader():获取全部响应头信息
      7.getResponseHeader():可根据需要查看响应头信息

      监听服务器

      onreadystatechange是XMLHttpRequest的一个重要属性，当readyState改变时，就会调用该函数。

      跨域

      一个域名地址的组成包括：
      　http://　　www . abc.com:　　8080 /　　sss.html
      　协议 　　子域名 　主域名　　端口号　　请求资源地址
      只要协议，子域名，主域名和端口号有一个不一样，就属于跨域。javascript出于安全的考虑，不允许跨域调用其他页面的数据。但有时又需要跨域获取数据，有三种操作方法：
      1.代理，前端访问后端，后端进行跨域访问
      2.JSONP：只能用于解决get请求，不支持post请求。
      　其基本思路：因为javascript可以跨域引用，通过跨域引用javascript文件获取数据
      3.XHR2：IE10一下不支持，最方面，只需在服务端进行小改动。
